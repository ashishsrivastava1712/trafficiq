Why Extract from Timestamp?

Raw timestamp is useless for ML

A model cannot learn from "2017-08-14 08:00:00" directly. Breaking it into hour=8, dow=0 (Monday), month=8 gives the model meaningful numerical patterns.

Temporal signals found in data

Analysis showed clear daily cycles (rush hours), weekly cycles (weekends), and seasonal cycles (summer peaks). Each needs its own feature.

2. Cyclic Feature Encoding (Key Innovation)
Why Sin/Cos Encoding? — The Cyclicality Problem
❌ Problem with Raw Hours
Hour 23 and Hour 0 are only 1 hour apart in reality, but numerically they are 23 apart.

Model thinks midnight (23→0) is a huge jump — but it's actually continuous!
|23 - 0| = 23 ← WRONG!
Actual distance = 1
✅ Sin/Cos Solution
Encode hour as a point on a circle. Hour 23 and Hour 0 are adjacent on the circle — distance = small!

This preserves the cyclical nature of time.
hour_sin = sin(2π × h/24)
hour_cos = cos(2π × h/24)

3. Domain Knowledge Features
🚗
rush_hour
1 if hour in [7,8,9,16,17,18] else 0
Binary flag for rush hours. Most important engineered feature — directly captures when commuters are on roads.
Importance: 22.3% — #2 Feature!
🏖️
is_weekend
1 if dow >= 5 else 0
Weekend flag. Traffic drops 35-40% on weekends as commuter traffic disappears. Strong negative predictor.
Importance: 9.9%
🌙
night
1 if hour in range(0,6) else 0
Night flag (12am-6am). Traffic consistently under 500 vehicles/hr. Helps model identify free-flow periods.
Importance: 3.1%
